# Semana 11 - Entrega incremental del sitio/proyecto

## Nombre de la tarea en Moodle
Semana 11 - Entrega incremental: depuracion y control de flujos en JavaScript

## Descripcion para publicar
En esta actividad cada estudiante o equipo debe entregar una version funcional del incremento semanal de su sitio/proyecto. El incremento consiste en un modulo de inscripcion a talleres comunitarios construido con HTML, CSS y JavaScript. El archivo `js/app.js` contiene errores intencionales que deben ser identificados y corregidos aplicando depuracion sistematica desde Visual Studio Code y la consola del navegador.

La entrega debe demostrar funcionalidad, integracion correcta, validaciones operativas, calidad visual y usabilidad basica.

## Producto a entregar
Un archivo ZIP con la carpeta completa del proyecto corregido, que incluya:

1. Codigo fuente funcional.
2. Checklist de requisitos diligenciado.
3. Matriz de pruebas diligenciada.
4. Bitacora de depuracion diligenciada.
5. Capturas de pantalla del sistema funcionando.
6. Capturas de la consola sin errores.
7. Captura de `tests.html` con pruebas aprobadas.
8. Changelog del incremento semanal.

## Modalidad
Individual o grupos de 2 a 4 estudiantes, segun disponibilidad de equipos y orientacion docente.

## Restricciones
- No se debe entregar solo una captura de pantalla.
- No se debe borrar todo el codigo y reemplazarlo sin explicar el proceso.
- No se debe entregar el proyecto con errores visibles en consola.
- No se debe omitir la bitacora ni el checklist.

## Criterios generales
- Funcionalidad demostrable.
- Integracion correcta de HTML, CSS y JavaScript.
- Validaciones operativas.
- Evidencias de pruebas.
- Calidad visual y usabilidad basica.
