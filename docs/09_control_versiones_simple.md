# Control de versiones simple

Si el grupo no usa Git, debe aplicar una estrategia basica para evitar perdida de informacion.

## Recomendacion de nombres
- `proyecto_semana11_v0_base.zip`
- `proyecto_semana11_v1_corregido.zip`
- `proyecto_semana11_v2_final.zip`

## Registro de cambios
| Version | Fecha | Cambio principal | Responsable |
|---|---|---|---|
| v0.1 | | Proyecto base recibido | |
| v0.5 | | Primeros errores corregidos | |
| v0.8 | | Validaciones completadas | |
| v1.0 | | Entrega final | |

## Buenas practicas
- No trabajar todos sobre el mismo archivo sin acordar turnos.
- Guardar copia antes de cambios grandes.
- Probar despues de cada correccion.
- Documentar cambios importantes en `CHANGELOG_INCREMENTO.md`.
