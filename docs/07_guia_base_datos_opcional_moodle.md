# Base de datos opcional en Moodle

La tabla de planeacion menciona que esta actividad puede complementarse con una Base de datos en Moodle. Esto no significa necesariamente usar una base de datos real como MySQL. Puede usarse la actividad Database de Moodle para registrar y compartir las entregas.

## Opcion A: Entrega solo por Tarea
Cada estudiante o equipo sube su ZIP a la actividad Tarea en Moodle.

## Opcion B: Tarea + Base de datos Moodle
Ademas del ZIP, cada estudiante o equipo crea una entrada en la base de datos del curso.

### Campos sugeridos para Moodle Database
1. Nombre del proyecto.
2. Integrantes.
3. Modalidad: individual / grupo.
4. Descripcion del incremento implementado.
5. Lista de validaciones realizadas.
6. Error mas importante corregido.
7. Captura del sitio funcionando.
8. Captura de consola sin errores.
9. Captura de pruebas automaticas.
10. Archivo ZIP o enlace de entrega.
11. Reflexion tecnica.

## Base de datos local opcional
El proyecto usa `localStorage` como almacenamiento local del navegador. Esto permite simular persistencia de datos sin instalar servidores ni motores de base de datos.

## Evidencia esperada si se usa Database
- Entrada creada en Moodle.
- Informacion completa y clara.
- Archivo o enlace funcional.
- Capturas visibles.
